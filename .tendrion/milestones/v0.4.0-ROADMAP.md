## Key Findings and Issues

### 1. Matrix4::from_translation had transposed column placement
The single most impactful bug. Translation was placed in row 3 of columns 0-2 instead of column 3, silently creating a projective transform that corrupted all extrusion-based geometry. This caused the `MissingPolygon` error that failed every boolean test involving `to_polygon()`. It had been hiding because most code paths didn't exercise the specific combination of translation + polygon extraction. Fix: one 2-line swap in `monstertruck-math/src/types.rs`.

### 2. Codex reviewer caught a fundamentally unimplementable plan (twice)
The coincident-face pre-classification approach for boolean repair was rejected across 2 review rounds with high-confidence blockers. The core issue: `divide_faces` creates new face identities via `Face::try_new`, so original shell face indices become meaningless after division. The reviewer correctly identified that no mapping strategy could bridge this identity gap. The approach was abandoned entirely in favor of diagnostic-only logging + resilient fallback defaults -- simpler and more correct.

### 3. Shell welding needed edge splitting, not just vertex merging
`weld_compressed_shell` merged coincident vertices but didn't split edges passing through those merged intermediate vertices. This left `adjacent_cubes_or` producing 2 shells instead of 1 -- faces from different input shells couldn't share sub-edges. Fix: `split_edges_at_intermediate_vertices()` detects edges with interior welded vertices, splits them using `Cut`, and deduplicates.

### 4. Edge deduplication matched wrong geometry by endpoints alone
`has_edge_between_points()` in the loops_store compared only endpoint positions, causing boundary edge segments to be falsely matched as duplicates of intersection curves. This broke `crossing_edges`. Fix: added midpoint verification -- after matching endpoints, compare the existing edge's curve midpoint with the intersection polyline's midpoint to distinguish geometrically different curves sharing endpoints.

### 5. punched_cube timeout from two compounding O(n^2) bottlenecks
The boolean pipeline timed out on revolution surfaces due to: (a) hill-climbing optimization re-converting intersection curves via Newton's method on every `evaluate` call instead of pre-converting once, and (b) `try_cap_shell_with_existing_surfaces` triangulating the entire shell per candidate face. Pre-converting known faces before the optimization loop and removing redundant triangulation from capping brought execution from timeout to ~4 seconds.
